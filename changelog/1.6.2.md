# v1.6.2

## [1.6.2] - 2026-03-08

### Fixed — Critical Financial Logic

- **XIRR SIP Cashflow Exclusion (P0):** `buildCashFlows()` in `portfolio.js` now correctly includes `sip_config` transactions (previously only matched literal `'sip'` type which no SIP ever had). XIRR calculations now account for all SIP investments.
- **Peer Lag Alert (Rule D) Never Fired (P0):** `runInsightAlerts()` Rule D was reading `category` and `returns1Y` from `allMfFunds` which only contains `schemeCode` and `schemeName`. Now correctly reads `subCategory` from `MFDB.getFund()` and retrieves ranked peers via `MFDB.getPeers()`.
- **SIP Rows Showed "NaN" Units in Transaction History (P1):** `renderTransactionHistory()` now detects `sip_config` type entries, falls back to `startDate` for date resolution, and displays `₹x,xxx/mo` in the units column instead of `NaN`.
- **`getCagrForYears` Rejected Valid Funds (P1):** Minimum data guard changed from `252 * years` (trading days) to `365 * years` (calendar days) to match the actual data density returned by mfapi.in.
- **Large & Mid Cap Funds Mis-classified (P1):** `SCHEME_CATEGORY_TO_LIVE_FUNDS` mapping corrected — `'Equity Scheme - Large & Mid Cap Fund'` now maps to its own distinct `'Large & Mid Cap'` bucket instead of `'Large Cap'`.
- **XIRR Duplicate Implementation Removed (P2):** `calcXIRR()` in `utils.js` deleted; `computeXIRR()` in `portfolio.js` is now the single canonical implementation (it also had the missing `-0.999` rate clamp guard).
- **XSS: `peer.schemeName` Injected Raw into innerHTML (P2):** Wrapped with `escapeHtml()` in `ui.js` peer ranking renderer.

### Fixed — Performance & Dead Code

- **Hardcoded 80ms Delay per Peer Fund Removed (P2):** `getPeerRanking()` discovery loop no longer sleeps 80ms between each fetch. Previously added up to ~9.6 seconds of pure synthetic wait for 120 funds.
- **Peer DB Cache Now Active (P1):** `fetchCategoryPeers()` now reads from `MFDB.getPeers()` before hitting the network and writes results back via `MFDB.setPeers()` after a successful fetch. Cache is valid for 24 hours.
- **Dead Code Removed:**
  - Empty `try {} catch {}` ghost block in `findTrueBestPeer()` (`api.js`)
  - `fetchTERFromGithub()` stub deleted — was fetching CSV but always returning `null`
  - Stale V1 schema constants (`MFAPP_DB_VERSION = 1`, `MFAPP_STORE = 'marketData'`) removed from `advisor-db.js`
  - `window.runDailySync` unused alias removed from `data-manager.js`
- **`dev-diagnostics.js` Now Dev-Only:** Health check script is gated behind `location.hostname === 'localhost'` and will not run in production.

### Implemented — Robo-Advisor Engine (P0)

- **Real Advisor Engine Worker:** `engine-worker.js` completely rewritten. Replaced 100% hardcoded mock results with real computations:
  - CAGR (1-year, 3-year, 5-year)
  - Annualised volatility (standard deviation of log-daily returns × √252)
  - Sharpe ratio (risk-adjusted: CAGR vs 6.5% India T-bill risk-free rate)
  - Maximum historical drawdown
  - Peer ranking sorted by cagr1y descending
  - Scored **HOLD / MONITOR / SWITCH** recommendation with written reasoning
- **`app.js` Peer Fetch Fixed:** Advisor UI now fetches real category peers from `MFDB.getPeers(subCategory)` instead of hardcoded mock entries.

### DevOps

- **`node_modules` Untracked from Git:** `git rm -r --cached node_modules/` executed. `.gitignore` already contained the rule.
- **`package.json` Updated:** Added `serve ^14.2.4` to `devDependencies`; version bumped to `1.6.2`.
