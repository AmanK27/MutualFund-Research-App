# v1.0.0 — Initial Release

**Date:** 2026-03-01

## Overview
First complete build of the MutualFund Research App — a zero-backend, single-page application for mutual fund research running entirely in the browser.

## Features

### Core Dashboard (Phase 1)
- **Search by Scheme Code** — Enter any AMFI scheme code to fetch live data from the free public API `https://api.mfapi.in/mf/{schemeCode}`
- **Fund Metadata** — Displays fund name, house, category, and scheme type
- **Financial Metrics** — Calculates and displays:
  - Latest NAV with date
  - CAGR for 1-Year, 3-Year, 5-Year, and Max periods
  - Annualised Standard Deviation (Volatility)
- **Interactive NAV Chart** — Chart.js-powered line chart with gradient fill, time-scale axis, and 1Y/3Y/5Y/Max range toggles
- **Watchlist** — Add/remove funds with `localStorage` persistence across sessions
- **Premium UI** — Dark-mode glassmorphism design with micro-animations, responsive layout, and Google Fonts (Inter)

### Category Navigation & Data Table (Phase 2)
- **Sidebar Navigation** — 4 collapsible fund categories: Hybrid Funds, Equity Funds, Debt Funds, Passives
- **15 Subcategories** — Conservative Hybrid, Balanced Advantage, Aggressive Hybrid, Arbitrage, Large Cap, Mid Cap, Small Cap, Flexi Cap, ELSS, Liquid, Money Market, Corporate Bond, Gilt, Index Funds, ETFs
- **Sortable Data Table** — 7 columns (Fund Name, NAV, 1Y CAGR, 3Y CAGR, Volatility, Sharpe Ratio, AUM) with click-to-sort ascending/descending and ▲/▼ indicators
- **Color-Coded Metrics** — Green for positive returns, red/amber for risk thresholds
- **Fund Drill-Down** — Click any table row to load real-time fund data from API
- **60+ Mock Funds** — Realistic prototype data across all subcategories

## Bug Fixes Applied
| # | Bug | Fix |
|---|-----|-----|
| 1 | `showState()` used `style.display = ''` — elements stayed hidden due to CSS `display:none` | Changed to explicit `'block'` |
| 2 | Invalid scheme codes returned `SUCCESS` with empty data — app crashed | Added checks for empty `data[]` and missing `meta.scheme_name` |
| 3 | `catch {}` without parameter — not compatible with older browsers | Changed to `catch (e) {}` |
| 4 | No protection against rapid double-clicks on Search | Added `isLoading` guard + disabled button |
| 5 | CORS error message unhelpful when opened via `file://` | Added descriptive error suggesting local HTTP server |
| 6 | Chart.js tooltip date formatting broken | Fixed `parsed.x` access instead of `instanceof Date` check |
| 7 | Empty NAV data after filtering caused crash | Added early guard with toast error |
| 8 | Fund names with `&` broke inline `onclick` handler | Replaced with `data-*` attributes + event delegation |

## Tech Stack
- HTML5, CSS3, Vanilla JavaScript
- Chart.js v4.4.1 (CDN)
- chartjs-adapter-date-fns v3.0.0 (CDN)
- Google Fonts (Inter)
- Browser `localStorage` for state persistence
