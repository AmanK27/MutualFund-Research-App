# v1.5.4

## [1.5.4] - 2026-03-02

### Fixed
- **AMFI Parsing Split Error:** Resolved a critical syntax error where the proxy response was inadvertently split using an escaped `\\n` instead of a literal `\n`, causing the entire 1.6MB file to be evaluated as a single unmatchable line in production arrays.
- **Fund Name Matching:** Upgraded the "Direct Plan - Growth" filter to be strictly case-insensitive (`.toUpperCase()`), ensuring hundreds of valid funds stylized in ALL CAPS by AMFI are not wrongfully discarded from the tables.
